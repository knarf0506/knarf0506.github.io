2.1 安裝 Ansible

  在本文中以 CentOS 7 為例，安裝管理主機及遠端主機。

2.1.1 管理主機

  安裝 Ansible。

sudo yum install epel-release
sudo yum install ansible

  設定 Ansible 管理主機和遠端主機連接。

# 產生 SSH 金鑰
ssh-keygen

# 檢查 SSH 公鑰(id_rsa.pub) 及私鑰(id_rsa)
cd .ssh/
ll

# 複製管理主機的 SSH 金鑰到遠端主機，這樣管理主機之後透過 SSH 連線至遠端主機就不需要密碼
ssh-copy-id -i ./id_rsa.pub remoteserver

  驗證 SSH 設定：在管理主機執行 SSH 指令遠端到遠端主機，若不需要密碼代表設定成功。

ssh remoteuser@remoteserver

2.1.2 遠端主機

  CentOS 7 需要確保啟動 SSH 服務，並請安裝 Python 版本 2.4 以上。

2.2 管理主機設定

  在上述中，管理主機透過 SSH 服務可以直接連線到遠端主機，但並不代表管理主機已經可以部署遠端主機。因此在管理主機上需要設定主機目錄設定檔。

2.2.1 預設設定檔

  在 Ansible 中，預設的 config file 在：/etc/ansible/ansible.cfg，可以透過以下指令查看。

ansible --version

# 返回結果

  configured module search path = [u'/root/.ansible/plugins/modules', u'/usr/share/ansible/plugins/modules']

2.2.2 撰寫設定檔

  如果需要自己撰寫設定檔，則目錄下需要包含 ansible.cfg 及 hosts。

  • Lesson1
    • ansible.cfg
    • hosts

  打開：/Lesson1/ansible.cfg。

[defaults]

inventory = ./hosts # 預設路徑是 /etc/ansible/hosts
host_key_checking = False # 多次連線情況下可能會報錯
private_key_file = /root/.ssh/id_rsa # 設定私鑰
log_path = ./log/ansible.log # 執行 ansible 指令時儲存 log 至此檔案

  打開：/Lesson1/hosts。

[node]

10.8.21.212
10.8.21.213

  在 /Lesson1 執行以下指令：

ansible all -m ping # 對所有 hosts 的 group 執行 ping 模組

# 返回結果

10.8.21.212 | SUCCESS => {
    "ansible_facts": {
        "discovered_interpreter_python": "/usr/bin/python"
    },
    "changed": false,
    "ping": "pong"
}
10.8.21.213 | SUCCESS => {
    "ansible_facts": {
        "discovered_interpreter_python": "/usr/bin/python"
    },
    "changed": false,
    "ping": "pong"
}

  在 /Lesson1 執行以下指令也可達到相同結果：

ansible node -m ping # 對 hosts 的 group node 執行 ping 模組

# 返回結果

10.8.21.212 | SUCCESS => {
    "ansible_facts": {
        "discovered_interpreter_python": "/usr/bin/python"
    },
    "changed": false,
    "ping": "pong"
}
10.8.21.213 | SUCCESS => {
    "ansible_facts": {
        "discovered_interpreter_python": "/usr/bin/python"
    },
    "changed": false,
    "ping": "pong"
}

  hosts 也可以修改為：

[node]

10.8.21.21[2:3] # 類似正則表達式的寫法，程式會執行 10.8.21.212, 10.8.21.213

  如果要修改連接埠，可以在 host 後面加上冒號及連接埠：

[node]

10.8.21.212:5000
10.8.21.213

  如果對 2 個 host 做分組：

[node1]

10.8.21.212

[node2]

10.8.21.213

  執行以下指令只會回傳 group 為 node1 的結果：

ansible node1 -m ping # 對 hosts 的 group node1 執行 ping 模組

# 返回結果

10.8.21.212 | SUCCESS => {
    "ansible_facts": {
        "discovered_interpreter_python": "/usr/bin/python"
    },
    "changed": false,
    "ping": "pong"
}

  對 hosts 創造另一個 group，而這個 group 可以是另一個 group 的子成員。

[node1]

10.8.21.212

[node2]

10.8.21.213

[node:children]

node1
node2

  執行以下指令：

ansible node -m ping
