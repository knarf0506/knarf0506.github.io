2.1 安裝 Ansible

  在本文中以 CentOS 7 為例，安裝管理主機及遠端主機。

2.1.1 管理主機

  安裝 Ansible。

sudo yum install epel-release
sudo yum install ansible

  設定 Ansible 管理主機和遠端主機連接。

# 產生 SSH 金鑰
ssh-keygen

# 檢查 SSH 公鑰(id_rsa.pub) 及私鑰(id_rsa)
cd .ssh/
ll

# 複製管理主機的 SSH 金鑰到遠端主機，這樣管理主機之後透過 SSH 連線至遠端主機就不需要密碼
ssh-copy-id -i ./id_rsa.pub remoteserver

  驗證 SSH 設定：在管理主機執行 SSH 指令遠端到遠端主機，若不需要密碼代表設定成功。

ssh remoteuser@remoteserver

2.1.2 遠端主機

  CentOS 7 需要確保啟動 SSH 服務，並請安裝 Python 版本 2.4 以上。

2.2 管理主機設定

  在上述中，管理主機透過 SSH 服務可以直接連線到遠端主機，但並不代表管理主機已經可以部署遠端主機。因此在管理主機上需要設定主機目錄設定檔。

2.2.1 預設設定檔

ansible --version

2.2.2 撰寫設定檔

  如果需要自己撰寫設定檔，則目錄下需要包含 ansible.cfg 及 hosts。

  • Lesson1
    • ansible.cfg
    • hosts

  打開：/Lesson1/ansible.cfg。

[defaults]

inventory = ./hosts # 預設路徑是 /etc/ansible/hosts
host_key_checking = False # 多次連線情況下可能會報錯
private_key_file = /root/.ssh/id_rsa # 設定私鑰
log_path = ./log/ansible.log # 執行 ansible 指令時儲存 log 至此檔案
